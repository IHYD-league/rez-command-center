## 🔴 Critical — fix before other families touch the app

### 1. Dev-placeholder copy still ships in JSX
Four places where parents will read words like "prototype" / "TODO" /
"real build" — same class of bug as the *On this journey* card I
already scrubbed:

- `src/App.jsx:66`  — `"Pediatrician", value: "TODO: add number"` (seed `EMERGENCY`)
- `src/App.jsx:70`  — `"Allergies: TODO add"` (seed)
- `src/App.jsx:2809` — `"Prototype role-switcher · TODO: real auth + server-enforced access windows"` (visible UI hint)
- `src/App.jsx:5289` — `"Photos & videos uploaded for this build the year-long portfolio. (TODO: video playback + cloud storage in the real build.)"` (Portfolio footer)
- `src/App.jsx:6984` — `"TODO: ICS / Google Calendar / school schedule import"` (Calendar footer)

**Fix:** scrub each. Either ship the real thing or remove the
forward-looking copy. Memory rule
`feedback_no_dev_placeholder_copy.md` already covers this.

### 2. Hardcoded "Reznor" / "Mike" / "Krissie" in user-visible strings
The multi-family pivot blocker. Every one of these reads wrong for
any family that isn't the Lynches:

| Where | Line | String |
|---|---|---|
| Stat detail header | `App.jsx:3115` | "Every star Reznor banked since midnight" |
| Stat detail header | `App.jsx:3118` | "Every star Reznor could earn if he finished everything on today's list" |
| Mark-done tooltip | `App.jsx:6145` | "Mark done for Reznor (with photo proof if needed)" |
| N/A confirm | `App.jsx:6155` | "It won't show on Reznor's board…" |
| Approvals section title | `App.jsx:6819` | "Wishes from Reznor" |
| Edit gift confirm | `App.jsx:1919` | "Reznor's bank will go up by…" |
| Confirm dialogs | various | "Reznor's bank will drop by N stars" |
| Quick-mark error | `App.jsx:9191` | "Couldn't find Reznor's profile." |
| Wish status (kid view) | `App.jsx:4350` | `"Waiting for Mom/Dad ⏳"` |
| Onboarding | `OnboardingOverlay.jsx:15` | "Reznor's checklist + family notes" |
| Settings hint | `CustomizationHub.jsx:561` | "so Grandma and Mike can be different" |

**Fix pattern:** every visible "Reznor" → `kid?.name || "your kid"`.
Every "Mike" / "Krissie" → "Mom/Dad" or "a parent" depending on the
context. Helper: a `kidName(users)` accessor + a `parentLabel(role)`.

The seed array (`SEED_USERS`, `CHILD`, contacts) is fine to leave —
those don't ship to other families because they're only seeded into
the Lynch's row. Just the user-visible *strings* need it.

### 3. STATUS_META renders flat English in the kid's MissionCard
`STATUS_META[status].label` returns "Not started" / "Pending approval"
/ "Approved" / "Needs fix" — Spanish/Both modes don't translate
these. Same applies to the **DayBreakdown** *Pending approval*
section heading.

**Fix:** route those four labels through `i18nTOf(...)` with new
`STRINGS` keys, same as the nav.

## 🟡 Important — polish + multi-family edges

### 4. Most `<SectionTitle>` headers stay English
36 SectionTitle call sites; only ~5 went through i18n in the
Phase 1 sweep. Reznor's Stars / Missions / Today screens still mix
Spanish task names with English section headers ("Today's missions"
not "Misiones de hoy").

**Fix:** seed all `<SectionTitle>` strings into `STRINGS` and use
`i18nTOf` — same pattern as nav labels. Effort: ~1 hr.

### 5. `alert()` for upload failures (10 spots)
Every photo / cover / avatar upload failure pops a native
`alert("Photo upload failed: ...")`. Jarring on mobile + un-styled
+ untranslated. Errors during a long submit feel like the app crashed.

**Fix:** small `useToast()` hook — replaces `alert()` with a styled
amber pill at the bottom-of-screen. Same shape the sync-failed banner
uses. Effort: 1 file.

### 6. Custom tasks have no translation field
Phase 1 only translates seeded task IDs (`t_eng`, `t_bed`, etc.). A
parent's added "Tidy desk" stays English in both modes. Memory plan
already calls this out as Phase 2.

**Fix:** add a `nameEs` (or generic `nameI18n`) field on tasks +
schema. Editor in ManageTasks gets a small "Spanish name (optional)"
input under the title. Effort: schema migration + UI.

### 7. Sync error stickiness
If one row in a batch violates a constraint (the recent
`reward_requests`/`denied` case), the whole batch upsert aborts —
unrelated writes go down with it. The sticky red banner gives the
parent no recovery action except waiting / restarting.

**Fix:** per-row retries on `denied`-style coercible mismatches
(I added one normalizer in `toDb`; pattern can extend). Longer-term:
batch writes per-row instead of all-at-once.

### 8. Photo-storage orphan files
When a parent removes a proof photo or undoes a completion, the
storage object isn't deleted. Mike's bucket grows slowly with
unreferenced files.

**Fix:** Edge Function periodic cleanup OR `on delete cascade`
behaviour on the `completions.proof[]` path. Defer until storage
quota becomes a concern.

## 🟢 Watchlist — fine today, revisit later

### 9. Bundle size 964 KB minified (264 KB gzipped)
Large for a mobile React app. Mostly fine over Wi-Fi; perceptible on
3G. Could split BoardGame themes + SongLogger out.

### 10. ManageTasks rename input stays raw English
Correct — the input is the editable seed value, and the translated
display reads from the i18n map keyed by `task.id`. If a parent
renames a seeded task, the i18n map still wins. Acceptable.

### 11. Some `task.title` lookups fall through to raw English
For tasks NOT in `TASK_TITLES` (any custom task the parent added),
`i18nTitleOf(task)` falls back to `task.title`. Correct behaviour;
covered by Phase 2 of the multilingual roadmap.

## ✅ Already solid — verified during this pass

- **RLS coverage** — every sensitive table (completions, gifted_stars,
  rewards, reward_requests, redemptions, books, songs, song_plays,
  streaks, awards, tasks) has `enable row level security` + `force
  row level security`. Secondary tables (board_state, user_prefs,
  summer_quest_progress, album_photos, family_state) also have policies
  per their migrations.
- **Status enums vs DB constraints** — recent fixes:
  - `reward_requests.status` ⇄ `declined` (coerced from legacy `denied`
    via toDb normalizer)
  - `redemptions.status` correctly uses `denied`
  - `completions.status` allows `draft` after the 2026-06-12 migration
  - Memory `feedback_match_status_vocab_to_constraint.md` prevents
    future drift
- **Error boundary** — generic copy, "your data is safe" reassurance,
  one-tap Copy of the stack. No more "send to Dad".
- **Sync failed banner** — visible at top of viewport with row-level
  failure messages and Dismiss.
- **Sync status dot** — bottom-right ambient indicator (green / amber
  / red) tied to the debounce queue.
- **Approve/deny actor identity** — server-side trigger
  `enforce_actor_identity_trg` plus upsert-replay tolerance keeps the
  audit honest.
- **Photo lightbox** — every photo across the app is tappable to
  full-screen with zoom and Esc-to-close. Body scroll locks while open.
- **Edit-everywhere gift sheet** — tap any bonus row from any surface
  (Star Ledger, DayBreakdown, Portfolio, Earned-today, Done) → edit
  sheet with confirm-on-save. No data dead-ends.
- **Cover vs proof distinction** — TaskSheet reading flow has the
  "Also use as the book cover" checkbox.
- **Draft completions** — save partial work for long tasks (Drums) and
  reopen later. Doesn't pollute the Approvals queue.
- **Family-level languages** — toggle in More → Languages applies to
  every member's view (parent, kid, helper).
- **Privacy & Safety page** — surfaces family ID, member roster, what's
  stored, photo geo policy, sign-out. Multi-family trust artifact.
- **Data audit page** — runs nine invariant checks (bank math, orphan
  refs, ISO dates) and surfaces drift before it leaks.
